# План исправления критических ошибок

Дата: 2026-07-18  
Статус: В работе

## Обнаруженные проблемы

### 1. Поиск возвращает неверные документы при большом индексе
**Симптомы:** На 7 книгах работает корректно. На 100 книгах (135 069 чанков) находит документы, не содержащие искомое слово.  
**Гипотеза:** Пересечение данных в postings.idx при многократных вызовах grow() во время индексации, либо неверное восстановление блоков при загрузке lexicon.  
**Приоритет:** Критический

### 2. Подсчёт символов в чанках неверен
**Симптомы:** `char_count` в БД показывает количество байт, а не UTF-8 символов.  
**Причина:** Используется `content.size()` (байты) вместо подсчёта code points.  
**Приоритет:** Высокий  
**Статус:** ✅ Исправлено (2.8.2)

### 3. Чанкер разрывает слова
**Симптомы:** Чанк заканчивается на `ме-` — слово разорвано на границе чанка.  
**Причина:** Текущий алгоритм разбивает текст по длине в байтах, а не по предложениям. Не учитывает границы предложений при склеивании/разбиении.  
**Приоритет:** Высокий

### 4. doc_id начинается с 0
**Симптомы:** Первый документ имеет doc_id=0.  
**Причина:** `last_doc_id_` инициализировался нулём в `IndexWriter::open()`.  
**Приоритет:** Низкий  
**Статус:** ✅ Исправлено (2.8.1)

### Этап 2.8.3: Исправить чанкер — разбивка по предложениям (проблема 3)
**Новая логика (на основе Go-кода):**
1. Вход: текст книги с параграфами, разделёнными `\n`
2. Для каждого параграфа:
   - Если длина в UTF-8 символах > `max_chunk_size`:
     а) Разбить параграф на предложения (`splitSentences`)
     б) Группировать предложения в чанки до `opt_chunk_size`
     в) Если предложений нет (нет знаков препинания) — разбить по словам с ограничением `max_chunk_size`
     г) Одиночное предложение длиннее `max_chunk_size` остаётся как есть (не разрывается)
   - Если длина <= `max_chunk_size`:
     а) Попытаться склеить с текущим накопительным чанком
     б) Если сумма > `max_chunk_size` — сохранить накопленный, начать новый
     в) Если сумма >= `opt_chunk_size` — сохранить
     г) Если сумма < `min_chunk_size` — продолжить склеивать
3. Сохранить оставшийся чанк
